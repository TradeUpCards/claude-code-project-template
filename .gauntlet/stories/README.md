# Interview Stories

A growing set of reusable interview artifacts — both for graded checkpoints (assignments, demo Q&A) and prospective-employer conversations.

Each story is a single markdown file. Two shapes are supported, pick the one that fits the question:

| Shape | Use for | Template |
|---|---|---|
| **Design Defense** | "Why did you choose X over Y?" — technical-judgment questions | `_template-design-defense.md` |
| **STAR** | "Tell me about a time you..." — behavioral / experience questions | `_template-star.md` |

The two shapes share a common structure underneath: a concrete situation, a specific decision or action, measurable outcomes, and an honest tradeoff. The difference is which surface the interviewer is probing.

---

## How to use during an interview

1. **Skim the index below** before the call. Pick 3–4 stories you can lead with.
2. **Memorize the hook line** of each — that's the first sentence you say.
3. **Practice the 30-second spoken version aloud** at least once.
4. **Have the file open** during the call so you can land specific numbers and citations without guessing.

When asked a question:
- Answer it in one sentence first.
- Then offer a story: *"There's a concrete example of this from <project> — want me to walk through it?"*
- If yes, deliver the structured version.
